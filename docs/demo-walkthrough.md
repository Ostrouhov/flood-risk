# floodrisk — сценарий демо под защиту

> Пошаговый прогон UI для защиты: **что нажать → что видно → что сказать комиссии**.
> Запуск сервиса — см. [../README.md](../README.md). Открыть http://127.0.0.1:8000.
> Тезис: сервис строит карту **пространственной подверженности** затоплению по рельефу
> (вероятность ∈[0,1] на пиксель), это **НЕ прогноз во времени** — «будущее» = сценарий осадков.

Перед демонстрацией: сервер поднят, БД засеяна (`python -m floodrisk.db.seed`), модель `unet-v1`
загружается резидентно при старте. Первый расчёт «прогревает» модель — сделайте один холостой
прогон до защиты.

## 0. Открыть приложение (15 c)
- **Видно:** карта по умолчанию центрируется на валидированной зоне (Тулун); пунктиром — контуры
  валидированных мозаик (Тулун, Канск). Слева — панель: сценарий / модель / зона / агрегаты /
  горячие точки / валидация / экспорт.
- **Сказать:** «Вход — прямоугольник территории и метео-сценарий. Выход — растровая карта
  вероятности затопления, построенная U-Net по рельефу и сопутствующим открытым геоданным».

## 1. Базовый прогноз: Тулун, p100y (30 c)
- **Сделать:** убедиться, что зона внутри Тулуна (кнопка `↺` сбрасывает к валидированной зоне);
  выбрать сценарий **p100y**; нажать **Пересчитать**.
- **Видно:** overlay Viridis (фиолетовый→жёлтый = рост вероятности); панель «Агрегаты»
  (площадь, доля p>0.5 / p>0.8, средняя p); «Расчёт» (источник=mosaic, латентность).
- **Сказать:** «Расчёт ~1.5–2 с на CPU. Источник признаков — заранее собранная мозаика
  валидированного региона. Жёлтые зоны — поймы и низины, куда стекает вода».

## 2. Чувствительность к сценарию (20 c)
- **Сделать:** переключить сценарий **p1y → p10y → p100y** (overlay пересчитывается автоматически).
- **Видно:** агрегаты монотонно растут (средняя p ≈ 0.07 → 0.13 → 0.20; доля p>0.5 ≈ 3% → 11% → 16%).
- **Сказать:** «Сильнее сценарий ливня (реже повторяемость) → больше площади подвержено. Сценарий
  применяется пост-сдвигом логита β=log10(RP/10) — это инженерная эвристика, а не обученная
  во времени зависимость; честно проговариваю границу метода».

## 3. «Горячие точки риска» — actionable-слой (30 c)
- **Сделать:** нажать **Показать топ-зоны риска**.
- **Видно:** на карте — пронумерованные маркеры топ-кластеров; в панели — список «#1 · площадь ·
  p̄»; клик по строке/маркеру → зум к зоне + попап (площадь, средняя/макс вероятность).
- **Сказать:** «Система не только красит карту, но и выделяет **конкретные зоны** наибольшего риска —
  связные компоненты выше порога, ранжированные по площади×вероятности. Это то, с чего начинать
  реагирование».

## 4. Объяснимость: почему здесь риск (30 c)
- **Сделать:** режим **Объяснение** → кликнуть по «горячей» зоне на карте.
- **Видно:** панель топ-5 признаков с долей важности (обычно лидирует Землепокров/рельеф) + слой
  атрибуции (magma) накладывается на карту; переключение слоёв по клику в списке.
- **Сказать:** «Для любой точки — вклад признаков через Integrated Gradients (U-Net) /
  feature_importances (baseline). Ограничения метода — в README (раздел Объяснимость)».

## 5. Научная валидация: реальность S1 ↔ предсказание (40 c)
- **Сделать:** вернуться в **Просмотр**; нажать **Реальность S1 ↔ предсказание (шторка)**.
- **Видно:** шторка: слева предсказание, справа — реальная маска разлива 2019 по Sentinel-1
  (красным); в панели — IoU/F1/precision/recall на видимой зоне; разделитель перетаскивается.
- **Сказать:** «Сравниваю предсказание с **фактическим** разливом 2019 (Sentinel-1). Это честная
  проверка на реальном событии, а не только на отложенной выборке».

## 6. Суть M-2: U-Net ↔ baseline (30 c)
- **Сделать:** нажать **U-Net ↔ baseline (шторка)**.
- **Видно:** шторка двух предсказаний + таблица агрегатов; U-Net даёт сфокусированные поймы,
  baseline — размытее.
- **Сказать:** «Главный научный вопрос M-2: оправдана ли пространственная нейросеть против простого
  попиксельного леса. U-Net **направленно** лучше по всем метрикам, но 95% доверительные интервалы
  пересекаются → статистически значимого превосходства на одном событии нет. Узкое место — **данные**
  (одно событие, дисбаланс ~1.2%), не метод. Это сильный, честный результат — он показывает границы
  и путь развития (больше событий/регионов). Разброс между сидами измерен — см.
  [../reports/seed_variance.md](../reports/seed_variance.md)».

## 7. Экспорт (15 c)
- **Сделать:** **Экспорт ZIP**.
- **Видно:** ссылка на скачивание; в архиве `prediction.tif` (GeoTIFF) + `aggregates.geojson` +
  `report.pdf`.
- **Сказать:** «Результат воспроизводимо выгружается: растр в нативной проекции, агрегаты, PDF-отчёт».

## 8. Мультирегион: Канск (experimental) (20 c)
- **Сделать:** в поиске места ввести «Канск» → перелёт; выделить зону внутри контура Канска;
  Пересчитать; (опц.) шторка реальности.
- **Видно:** source=mosaic; шторка реальности помечена **экспериментально** (жёлтый кавеат).
- **Сказать:** «Перенос на другой регион (р. Кан) работает на тех же открытых источниках. Канск —
  демо: признаки корректны, но S1-лейблы не сверены с эталоном (пере-детекция) → в метрику M-2 не
  входит, и в UI об этом честно сказано».

## 9. Глобальный онлайн-режим (опц., 30 c, нужна сеть)
- **Сделать:** найти «Москва» (или любой город вне регионов); выделить небольшую зону (<30 км);
  Пересчитать.
- **Видно:** баннер «экспериментально»; признаки собираются на лету (Planetary Computer); дольше
  (до минуты), оверлей строится.
- **Сказать:** «По произвольному bbox признаки собираются онлайн из открытых каталогов. Это
  демонстрирует общность метода, но без локальной валидации — помечено экспериментальным».

## FAQ — короткие честные ответы комиссии
- **«На сколько метров поднимется вода?»** Вне скоупа. Модель даёт **факт затопления** (вероятность
  принадлежности к зоне разлива), не глубину. Глубина/динамика — гидродинамика, явный анти-гол.
- **«Когда затопит?»** Это не прогноз во времени. Выход — подверженность при **заданном сценарии**
  осадков (повторяемость 1/10/100 лет = насколько редкий и сильный один ливень).
- **«Почему M-2 fail — это плохо?»** Нет: результат валиден и задокументирован. Ограничение — объём
  данных (одно событие), а не реализация. Метод корректен, путь к улучшению ясен.
- **«Почему U-Net, а не таблица?»** Затопление пространственно (куда стекает вода зависит от
  соседей). Свёрточная сеть это учитывает, попиксельный лес — нет. Baseline нужен именно для этого
  сравнения.
- **«Воспроизводимость?»** Фиксированные сиды, lock-файлы, манифест с sha256, Docker; разброс метрик
  между сидами измерен ([../reports/seed_variance.md](../reports/seed_variance.md)).
