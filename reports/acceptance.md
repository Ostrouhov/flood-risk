## Багфикс препроцесса признаков + мультирегион (пост-Этап-6)

**Найден и исправлен критический баг сборки признаков.** `pipeline.run_preprocess` брал ОДИН
исходный тайл (`_first()`) вместо мозаики всех тайлов, перекрывающих сетку → DEM/slope/curvature
были заполнены лишь в **~5 %** площади Тулуна (остальное — нули/константы); модель v1 фактически
училась на вырожденных признаках. Фикс — `preprocess.mosaic_source_to_file` (склейка всех
пересекающих сетку тайлов DEM/WorldCover/JRC + обрезка по bbox; bbox-тег в имени файла против
коллизий между регионами при общем `data/raw/`). Регресс-тесты: `tests/test_preprocess_mosaic.py`.

**Последствия:** Тулун перестроен (DEM заполнен **98.6 %**), **U-Net+baseline переобучены на
корректных признаках**, M-2 переоценён (см. выше) — вердикт FAIL подтверждён уже на валидных данных.

**Мультирегион (демо):** добавлен 2-й регион **Канск (р. Кан, 2019)** — офлайн-мозаика признаков,
инференс выбирает регион по bbox (`service._resolve_features` перебирает `data/processed/*/features/
stack.tif`), на карте 2 контура покрытия, шторка «реальность S1 ↔ предсказание» работает для обоих.
Канск помечен **experimental**: признаки корректны, но S1-лейблы пере-детектированы (6.8 %, без
эталона EMS) → в M-2 не используется; в UI — честный кавеат. Москва/вне регионов — онлайн-режим.

## M-4 — ручной чек-лист UI

Прогон по HTTP-поверхности сервиса (uvicorn) на пилотной территории (Тулун, bbox
`100.64,54.54,100.89,54.71`); поведение DOM атрибуции дополнительно смоделировано HTML-парсером:

- [x] `p1y`/`p10y`/`p100y` — overlay меняется: mean_p 0.055 → 0.106 → 0.184; p>0.5 0 → 0.077 → 0.124.
- [x] Агрегаты возвращаются во фрагменте и согласованы с растром (FR-16 OOB swap).
- [x] PNG overlay перепроецирован в EPSG:4326 и отдаётся (`/runs/<id>/prediction.png`, 200 image/png).
- [x] Экспорт → ZIP (1.79 МБ): `prediction.tif` + `aggregates.geojson` + `report.pdf`; `HX-Trigger: download-ready`.
- [x] Режим «Объяснение» → панель топ-5 (Землепокров 47% … DEM 5%) + **слой атрибуции** (5 PNG, magma).
- [x] Тайминги без фриза > 2 с: predict 1.6–1.9 с, explain ~6 с (< NFR-3 15 с), export 0.7 с.
- [x] Ошибки чисто: out-of-coverage bbox → сообщение (не 500); unknown run export → «запуск не найден».

> **Найдено и исправлено при прогоне M-4 (2026-06-05):** в `explanation_panel.html` атрибут
> `data-attribution-layers` рендерился в двойных кавычках вокруг `| tojson` — внутренние `"`
> JSON-а обрезали значение до `[{` в браузере → `JSON.parse` падал (тихо, через try/catch в
> [app.js](../static/app.js)), и **слой важности не накладывался на карту**. Фикс: двойные кавычки →
> одинарные (`tojson` экранирует `'`). После фикса DOM-значение — валидный JSON из 5 слоёв.
> Ранее (Этап 5) проверялся, видимо, только серверный ответ, не DOM.
